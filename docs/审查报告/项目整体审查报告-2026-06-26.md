## 1. 项目概况

### 1.1 项目定位
Vue3 中后台前端平台脚手架 Monorepo，以 Git 模板仓库形式交付，面向团队复用。

### 1.2 技术栈（v2.0）
- **框架**：Vue 3.5+
- **组件库**：Element Plus 2.9+
- **样式方案**：Tailwind CSS 4（替代 UnoCSS）
- **HTTP 客户端**：ky 1.x（替代 axios）
- **服务端状态**：TanStack Vue Query 5
- **Mock 方案**：MSW 2.5+
- **构建工具**：Vite 8.0+
- **包管理**：pnpm 10.33.1 + catalog + overrides

### 1.3 仓库结构
```
vue-admin-monorepo/
├─ apps/vue3-app/              # Vue3 正式宿主应用
├─ packages/
│  ├─ design-tokens/           # 设计令牌
│  ├─ shared-utils/            # 通用工具
│  ├─ shared-i18n/             # 国际化
│  ├─ shared-service/          # 服务层
│  ├─ shared-ui/               # UI 组件
│  └─ shared-workflow/         # 工作流引擎
├─ docs/decisions/             # ADR 文档（9 份）
└─ 根配置文件
```

## 7. 发现的问题

### 7.1 轻微问题

#### 问题 1：STATUS.yaml 更新时间
- **位置**：`STATUS.yaml`
- **现状**：`updated_at: 2026-05-19`
- **建议**：v2.0 迁移完成后，应更新为 2026-06-26

#### 问题 2：pnpm.onlyBuiltDependencies 警告
- **位置**：根 `package.json`
- **现状**：`"pnpm": { "onlyBuiltDependencies": ["msw"] }`
- **说明**：pnpm 10.x 不再读取此字段，应迁移到 pnpm 设置文件
- **影响**：仅警告，不影响功能
- **建议**：迁移到 `.npmrc` 或 `pnpm-workspace.yaml` 的 `onlyBuiltDependencies` 字段

### 7.2 待完善项

#### 待完善 1：design-tokens Tailwind 主题导出
- **现状**：Tailwind 主题直接在 `apps/vue3-app/src/assets/main.css` 中定义
- **设计文档要求**：design-tokens 应导出 `./tailwind-theme` 子路径
- **影响**：低，当前实现可用，但不符合设计文档的模块化原则
- **建议**：将 Tailwind 主题配置迁移到 design-tokens 包，并通过 `./tailwind-theme` 导出

#### 待完善 2：shared-service HttpClient 注入机制
- **现状**：shared-service 已移除 axios，但未发现明确的 HttpClient 注入代码
- **设计文档要求**：shared-service 应通过注入的 HttpClient 实例发起请求
- **影响**：中，需要确认 API 模块是否实际使用了 HttpClient
- **建议**：检查 shared-service/src/api/ 目录下的实现，确保使用 HttpClient 接口

#### 待完善 3：Vue Query 实际使用
- **现状**：Vue Query 已注册，但未发现视图层使用 useQuery/useMutation 的示例
- **影响**：低，基础设施已就绪
- **建议**：在示例页面或后续开发中展示 Vue Query 的使用方式
